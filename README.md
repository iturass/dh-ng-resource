# dh-ng-resource

## 项目简介

`dh-ng-resource` 是一个用于归档和发布独立 HTML 页面的仓库。本项目不针对任何特定域名或网站，仅作为静态 HTML 资源的存储和展示空间。所有页面均为独立的 HTML 文件，可直接通过浏览器访问。

## 目录结构

```
dh-ng-resource/
├── pages/          # 存放各个独立 HTML 页面
├── assets/         # 页面所需的公共资源（CSS、JS、图片等）
└── README.md       # 本文件
```

- **pages/**：每个子目录或文件代表一个独立的 HTML 页面，页面之间互不依赖。
- **assets/**：存放页面共享的样式、脚本、字体、图片等静态资源。

## 页面归档说明

- 所有页面均为纯前端实现，不涉及后端服务。
- 页面内容可能包含文本、图表、交互组件等，具体以实际文件为准。
- 归档的页面可能来自不同时间、不同主题，彼此之间不存在内容关联。
- 页面链接、资源引用均使用相对路径，确保在本地或静态托管环境中正常访问。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/dh-ng-resource.git
   ```
2. 进入 `pages/` 目录，直接双击打开任意 HTML 文件即可在浏览器中查看。
3. 若需部署至静态服务器，将整个仓库内容上传至服务器根目录即可。

## 维护说明

- 本仓库由维护者不定期更新，新增或修改页面会直接提交至对应目录。
- 页面内容不会进行主动推广或链接交换，仅供浏览参考。
- 如发现页面显示异常或资源缺失，欢迎通过 Issues 反馈。
- 不接受外部提交内容，本仓库为个人归档用途。
